SQL Flow

Executes a query against SQL Server. Click the connection line to simulate a network failure.

Available Scenarios:

  • Roundtrip - Normal query execution returning a GUID and timestamp
  • Wrong Table - Query references non-existent table (generates SQL error span)
  • Wrong Column - Query references non-existent column (generates SQL error span)
  • Syntax Error - Malformed SQL syntax (generates parse error span)
  • Division Error - Division by zero (generates arithmetic error span)

Telemetry Generated:

Each request creates spans with db.system, db.operation, and error details when applicable.

Redis Flow

Performs cache operations against Redis. Click the connection line to simulate Redis being unavailable.

Available Scenarios:

  • Roundtrip - Normal SET/GET operation cycle
  • Missing Key - GET on non-existent key (returns null, no error)
  • Large Value - Store 10KB payload to simulate memory pressure
  • Expired Key - Key expires immediately after SET
  • Serialization Error - Simulates corrupt data during deserialization
  • Invalid Operation - Wrong data type operation (WRONGTYPE error)

Telemetry Generated:

Each request creates spans with db.system=redis, db.operation (GET/SET), and cache-specific attributes.

Simulation (Saga)

Generates rich, multi-stage traces simulating a distributed transaction across microservices.

Pipeline Stages:

  1. Validation - Schema, permissions, quota, and rate-limit checks
  2. Data Fetch - Parallel SQL queries (3) and Redis operations (4)
  3. Processing - Transform & enrich with retry logic (exponential backoff)
  4. Saga - Distributed transaction across microservices
  5. Finalization - Cleanup, audit logging, and metrics recording

Simulated Microservices:

  • order-service - Handles order creation and management
  • inventory-service - Manages stock levels and reservations
  • payment-service - Processes payment transactions
  • notification-service - Sends order confirmations and alerts

Service Attributes:

Each saga span includes service.name, service.version, and service.instance.id for realistic distributed tracing.
